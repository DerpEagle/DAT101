1. In part 1, is it possible to solve the problem using a single "for" loop?

    2. Ja, det er mulig å løse oppgaven med bare én “for”-loop, men det krever at man kombinerer både telling opp og telling ned inne i samme loop.

    Det kan bli mindre oversiktlig og vanskeligere å lese enn å bruke to separate looper.

    Derfor anbefales det å bruke to looper for klarhet og struktur.

  2. In part 2, can we create what is known as an "infinite" loop within the condition of the "while" loop? Describe what would or would not be necessary to achieve this. Provide a thorough explanation of your answer.

    3. Teoretisk kan man lage en infinite loop med while ved å lage en betingelse som aldri blir oppfylt.

    I oppgaven med gjetting vil loopen bare bli uendelig hvis tallet som skal gjettes aldri endres eller aldri kan bli korrekt.

    I praksis ønsker man ikke en uendelig loop, fordi programmet skal stoppe når riktig tall er gjettet.

    Så for å få en uendelig loop må man unngå å endre variabelen som sjekkes, men det er ikke nødvendig eller ønsket her.

  3. Discuss briefly how quickly or slowly a computer might be able to compute this, considering the number of guesses and the time it takes. How does the program behave while it is guessing? Can the program perform other tasks concurrently?

    4. Hastigheten avhenger av hvor mange mulige tall som skal gjettes og hvor tilfeldig gjettingen skjer.

    For små tall (1–60) går det veldig raskt, og datamaskinen gjetter nesten umiddelbart.

    For store tall (1–1 000 000) kan det ta lang tid fordi tilfeldige gjetninger kan kreve mange millioner forsøk.

    Under gjettingen blokkerer programmet i nettleseren, så det kan ikke gjøre andre oppgaver samtidig.

    JavaScript er enkelttrådet, så alt venter på at loopen skal fullføre.

  4. Explain the relationship between rows and columns in nested "for" loops. Can we, for instance, change the order, considering rows to be innermost or columns to be innermost, or does this not matter?

    5. I nested loops er den ytre loopen vanligvis rader, og den indre loopen kolonner.

    Dette betyr at for hver rad blir alle kolonner kjørt ferdig før neste rad starter.

    Man kan bytte om rekkefølgen, men da endrer det hvordan output organiseres – det vil si at man må tenke på hva som skal komme først.

    Rekkefølgen påvirker altså strukturen på det som skrives ut.
